STEP 3 — ZUSTAND STORES
=======================================================================

--- store/webhookStore.ts ---
  State: webhooks, selectedWebhook, loading, filters
  Actions:
    fetchWebhooks(filters?)     → GET /api/webhooks
    selectWebhook(event)
    addWebhook(event)           → prepend to array (called from socket)
    replayWebhook(id)           → POST /api/webhooks/:id/replay
    deleteWebhook(id)           → DELETE /api/webhooks/:id
    clearSelected()
    setFilters(filters)         → update filters + re-fetch

--- store/apiRequestStore.ts ---
  State: apiRequests, selectedRequest, loading, filters
  Actions:
    fetchApiRequests(filters?)  → GET /api/api-requests
    selectRequest(req)
    addApiRequest(req)          → prepend (from socket)
    sendProxyRequest(data)      → POST /api/api-requests/proxy
    replayApiRequest(id)        → POST /api/api-requests/:id/replay
    deleteApiRequest(id)        → DELETE /api/api-requests/:id
    clearSelected()
    setFilters(filters)

--- store/uiStore.ts ---
  State: activeTab, sidebarOpen, debugPrompt, debugPromptLoading
  Actions:
    setActiveTab(tab)
    toggleSidebar()
    generateDebugPrompt(type, eventId) → POST /api/ai-debug/prompt → set debugPrompt
    clearDebugPrompt()

STEP 4 — HOOKS
=======================================================================

--- hooks/useSocket.ts ---
  - Connect via socket.io-client to VITE_API_BASE_URL
  - "new_webhook" → webhookStore.addWebhook(event) + toast.success
  - "new_api_request" → apiRequestStore.addApiRequest(event) + toast
  - Handle connect/disconnect lifecycle
  - Return { connected: boolean }

--- hooks/useDebugPrompt.ts ---
  - Wrap uiStore.generateDebugPrompt
  - Return { prompt, loading, generate(type, eventId), clear }

STEP 5 — COMPONENTS
=======================================================================

Use lucide-react for all icons. TailwindCSS v4 only.

Key UI behaviors to implement (design the look yourself using the skill):

  StatusBadge.tsx { status: number }
    - Must be a pill/badge shape
    - Visually distinct between success (2xx), client error (4xx), server error (5xx)

  MethodBadge.tsx { method: string }
    - Must be a pill/badge shape
    - Each HTTP method (GET/POST/PUT/PATCH/DELETE) must be visually distinct

  ConnectionStatus.tsx
    - MUST show a live animated pulsing dot when socket is connected
    - MUST show a clearly different disconnected state
    - Fixed position in navbar

  TimestampDisplay.tsx { timestamp: string }
    - Use date-fns for formatting
    - Show relative time on hover (e.g. "2 minutes ago")

  JsonViewer.tsx { data: Record<string, unknown> | string; label?: string }
    - Collapsible JSON block, monospace font
    - Copy to clipboard button

  EventCard.tsx { event: WebhookEvent | ApiRequest; type: EventType; onClick }
    - Compact card: method badge, source/service, status badge, timestamp, responseTime
    - Must have a visually prominent indicator for failed events
    - Animated entrance on appear

  EventList.tsx { type: EventType }
    - Scrollable list of EventCards
    - Empty state: "No events captured yet. Waiting for incoming traffic..."
      with an animated indicator
    - Loading skeleton while loading: true

  EventDetailPanel.tsx { event: WebhookEvent | ApiRequest | null; type: EventType }
    - Sections: Overview, Request Headers, Payload, Response Body (ApiRequest only)
    - Action buttons: [Replay] [Generate AI Prompt] [Delete]
    - Placeholder when event is null

  DebugPromptModal.tsx
    - Full-screen overlay with generated prompt in a code block
    - [Copy Prompt] button + [Close] button
    - Escape key closes modal

  ProxyRequestForm.tsx
    - Fields: Method selector, Endpoint URL, Headers key-value editor, Payload textarea
    - JSON validation on payload with inline error
    - Loading state on submit

  FilterBar.tsx { type: EventType }
    - Source filter input, failed-only toggle, limit selector (25/50/100)
    - [Clear Filters] button

  Navbar.tsx
    - App name "DevProxy" with icon
    - Tab switcher: [Webhooks] [API Requests]
    - ConnectionStatus on right

  StatsSummary.tsx { type: EventType }
    - 4 stat cards: Total | Failed | Avg Response Time | Last Event Time
    - Numbers animate on change
