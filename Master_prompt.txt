MODULE 1: webhooks  (backend/src/modules/webhooks/)

--- webhooks.model.ts ---

Create a Mongoose model called `WebhookEvent` with this schema:

  interface IWebhookEvent {
    id: string                  // uuid, unique identifier
    source: string              // e.g. "razorpay", "stripe", "github" — parsed from URL
    method: string              // HTTP method: GET, POST, etc.
    url: string                 // full request URL
    headers: Record<string, string>
    payload: Record<string, unknown> | string
    status: number              // HTTP status returned to the provider
    responseTime: number        // in milliseconds
    timestamp: Date             // auto set to Date.now()
    failed: boolean             // true if status >= 400
  }

Add timestamps: true to the schema options.
Add an index on `timestamp` descending for fast queries.

--- webhooks.controller.ts ---

Implement these controller functions:

  1. interceptWebhook(req, res):
     - Capture method, url, headers, body from req
     - Parse source from URL (e.g. "/webhooks/razorpay" → "razorpay")
     - Record start time, then forward request to BACKEND_TARGET_URL using axios
     - Record responseTime
     - Save full WebhookEvent to MongoDB
     - Emit event via Socket.io: socket.emit("new_webhook", savedEvent)
     - Respond to the original caller with the proxied response
     - Mark failed: true if status >= 400

  2. getAllWebhooks(req, res):
     - Query all WebhookEvents sorted by timestamp descending
     - Support optional query params: ?source=razorpay&failed=true&limit=50
     - Return paginated JSON

  3. getWebhookById(req, res):
     - Find one WebhookEvent by its uuid `id` field
     - Return full document including headers + payload

  4. replayWebhook(req, res):
     - Find event by id
     - Re-POST the stored payload + headers to BACKEND_TARGET_URL
     - Save a NEW WebhookEvent record with same data but new timestamp
     - Emit "new_webhook" via socket
     - Return the new event

  5. deleteWebhook(req, res):
     - Delete event by id

--- webhooks.routes.ts ---

  POST   /webhooks/:source         → interceptWebhook
  GET    /webhooks                 → getAllWebhooks
  GET    /webhooks/:id             → getWebhookById
  POST   /webhooks/:id/replay      → replayWebhook
  DELETE /webhooks/:id             → deleteWebhook

MODULE 2: api-requests  (backend/src/modules/api-requests/)

--- api-requests.model.ts ---

Create a Mongoose model called `ApiRequest` with this schema:

  interface IApiRequest {
    id: string                    // uuid
    method: string
    endpoint: string              // e.g. "https://api.razorpay.com/v1/payments"
    requestHeaders: Record<string, string>
    requestPayload: Record<string, unknown> | string
    responseStatus: number
    responseBody: Record<string, unknown> | string
    responseTime: number          // ms
    timestamp: Date
    failed: boolean               // true if responseStatus >= 400
    service: string               // parsed from endpoint hostname
  }

Add timestamps: true and index on timestamp descending.

--- api-requests.controller.ts ---

Implement these controller functions:

  1. proxyApiRequest(req, res):
     - Accept { method, endpoint, headers, payload } in req.body
     - Use axios to forward the request to `endpoint` with provided headers + payload
     - Record start/end time for responseTime
     - Save ApiRequest document to MongoDB
     - Emit "new_api_request" via Socket.io
     - Return { request: savedDoc, response: { status, body } }
     - Mark failed: true if status >= 400

  2. getAllApiRequests(req, res):
     - Return all ApiRequests sorted by timestamp descending
     - Support ?service=razorpay&failed=true&limit=50

  3. getApiRequestById(req, res):
     - Return single ApiRequest by uuid id

  4. replayApiRequest(req, res):
     - Find by id, re-forward the original request through proxyApiRequest logic
     - Save new record, emit socket event, return new record

  5. deleteApiRequest(req, res):
     - Delete by id

--- api-requests.routes.ts ---

  POST   /api-requests/proxy         → proxyApiRequest
  GET    /api-requests               → getAllApiRequests
  GET    /api-requests/:id           → getApiRequestById
  POST   /api-requests/:id/replay    → replayApiRequest
  DELETE /api-requests/:id           → deleteApiRequest

MODULE 3: ai-debug  (backend/src/modules/ai-debug/)

--- ai-debug.controller.ts ---

  1. generateDebugPrompt(req, res):
     - Accept in req.body:
         { type: "webhook" | "api_request", eventId: string }
     - Fetch the event from the appropriate collection by eventId
     - If type === "webhook", fetch from WebhookEvent
     - If type === "api_request", fetch from ApiRequest
     - Build and return a structured AI debugging prompt string in this exact format:

       ---
       A [webhook/API] request failed.

       [If webhook:]
       Source: <source>
       Method: <method>
       URL: <url>

       [If api-request:]
       Endpoint: <endpoint>
       Method: <method>
       Service: <service>

       Payload:
       <JSON.stringify(payload, null, 2)>

       Headers:
       <sanitized headers — mask Authorization values as "Bearer ***">

       Response Status: <status>
       Response Body: <JSON.stringify(responseBody, null, 2)>
       Response Time: <responseTime>ms

       You are a Node.js backend expert. Identify the root cause of this failure
       and suggest a precise fix with corrected code. Be specific about which field
       or header is wrong and why.
       ---

     - Return { prompt: string }

--- ai-debug.routes.ts ---

  POST   /ai-debug/prompt   → generateDebugPrompt

IMPORTANT IMPLEMENTATION RULES
=======================================================================

1. NEVER use `any` type anywhere — use `unknown` and narrow properly
2. ALL async route handlers must be wrapped in asyncHandler()
3. ALL controllers must use try/catch or asyncHandler — no unhandled rejections
4. ALL Mongoose queries must be typed with the interface generic
5. Socket.io instance must ONLY be initialized once (singleton via shared/socket.ts)
6. Frontend components must NEVER call axios directly — always through Zustand actions
7. JSON payloads in the UI must always be safely parsed with try/catch before display
8. Authorization header values must ALWAYS be masked in the AI debug prompt
9. All dates must be formatted using date-fns — never use raw Date.toLocaleString()
10. TailwindCSS v4 only — do not use v3 syntax or @apply in component files
